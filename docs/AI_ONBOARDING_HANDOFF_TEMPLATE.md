# AI Onboarding Handoff Template

Copy this structure into a target repo handoff when an AI onboarding task must
resume cleanly without chat history.

```text
Action
- Goal:
- Work completed:
- Work not completed:

Proof
- Repo root:
- Remote URL:
- Branch:
- Commit:
- Dirty worktree:
- Files read:
- Files changed:
- Files created:
- Files intentionally not touched:
- Validation commands:
- Validation exact results:
- GitHub governance proof:
- External-system proof:

Status Split
- Onboarding docs:
- Diagnostic scripts:
- Diagnostic tests:
- Local validation:
- CI validation:
- External systems:
- Repo config:
- Caller workflow:
- Protected branch/ruleset:
- Required checks:
- Canary PR:
- Repo GitHub governance:

Risk
- Unresolved gaps:
- Missing proof:
- Known fragile areas:
- Stop conditions hit:
- Owner-scoped items:

Next
- Exact next safe action:
- Exact next command:

Unknowns
- List exact unknowns, or None.
```
